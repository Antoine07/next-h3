# TP — Créer et comprendre la base de données d’une application de quiz

##  Objectifs pédagogiques

## Indications

Vérifiez avant de commencer le TP que vous avez bien une connexion à la base de données

```
mysql -u root -p
```

Créez un fichier `init_quiz_app.sql` pour mettre dans ce fichier le script permettant de créer les tables de la base de données

Une fois dans le dossier insérez les données en base de données.

```sql
mysql -uroot -psecret quiz_app < init_quiz_app.sql
```

##  Contexte

Vous travaillez sur une application de **quiz en ligne**.

Un quiz :

* appartient à un utilisateur
* contient plusieurs questions
* chaque question a plusieurs choix (QCM)
* les utilisateurs peuvent laisser des retours (feedback)
* les quiz peuvent être associés à plusieurs tags

##  Création de la base (SQL)

### Créer la base de données

```sql
CREATE DATABASE quiz_app
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

Puis :

```sql
USE quiz_app;
```

---

### Créer les tables (structure)

Créer les tables suivantes :


* `quizzes`
* `questions`
* `choices`

( Facultatives )

* `users`
* `feedbacks`
* `tags`
* `quiz_tags`

👉 **Consignes** :

* chaque table a une clé primaire
* les relations utilisent des clés étrangères
* les textes sont en UTF-8

---

##   Insertion de données simples

### Insérer des données

Insérez :

* 2 utilisateurs
* 2 quiz
* au moins :

  * 1 question par quiz
  * 2 choix par question
  * 1 tag par quiz

> Les textes peuvent être simples (ex. “Oui / Non”).

---

## Vérifications

###  Vérifier les relations

Exécutez les requêtes suivantes :

```sql
SELECT * FROM quizzes;
SELECT * FROM questions;
SELECT * FROM choices;
```
